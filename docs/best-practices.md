# Best Practices for Google Antigravity

## ✅ DO

- **Write clear, specific task descriptions** — The more precise your instructions, the better the agent performs
- **Use development modes appropriately** — Start with Agent-Assisted mode until you're comfortable
- **Review artifacts carefully** — Check implementation plans before letting agents proceed
- **Provide feedback early** — Redirect agents mid-task rather than waiting for completion
- **Break complex tasks into steps** — Use task groups for multi-step projects
- **Set up rules and workflows** — Define project-specific constraints for consistent results

## ❌ DON'T

- **Don't use vague instructions** — "Make it better" gives poor results
- **Don't skip artifact review** — Always verify the agent's plan before approval
- **Don't ignore test results** — Agents can make mistakes; validate their output
- **Don't mix unrelated tasks** — Keep tasks focused for better results

## Tips for Effective Agent-Driven Development

### Start Small

Begin with well-defined, scoped tasks. As you learn how agents work, gradually increase task complexity.

### Leverage Multi-Model Support

Use faster models (Gemini Flash) for simple tasks and more capable models (Gemini Pro, Claude Opus) for complex reasoning.

### Use Rules for Consistency

Define project rules to ensure agents follow your coding standards consistently:
- Language preferences
- Formatting requirements
- Testing expectations
